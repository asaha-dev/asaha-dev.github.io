from ReAct to agent harness: how to think about the shift#llm #systems #leadership
This is the first post in a series on LLM systems: why the industry is moving away from "just give the model more context" and toward explicit memory formation and orchestration systems. The series goes deeper in context and memory, building systems around transformers, and Claude Code — a systems architecture deep dive. ReAct (Yao et al., 2022) was widely popular in 2022–2023 for building AI applications on large language models. The core idea was neat: let the model interleave reasoning traces with tool calls in one flat loop. Instead of forcing the LLM to either think internally and answer (User → LLM → Answer) or call tools immediately (User → LLM → Tool → LLM → Answer), you alternate between thought, action, and observation — possibly repeating — before arriving at an answer. For example: Question: Who is the current CEO of Nvidia? Thought: I need to find who the CEO is. Action: Search(&quot;CEO Nvidia&quot;) Observation: Jensen Huang is the founder and CEO of Nvidia. Thought: I found the answer. Answer: Jensen Huang This became the blueprint for most modern agents. ReAct became the dominant pattern for early LangChain agents and tool-calling implementations. LangGraph later emerged as a more structured alternative — adding explicit state and control flow in response to the flat loop's limitations at scale. An agent harness is the systems idea that wraps it: a deterministic orchestration layer around a non-deterministic model that explicitly owns context, control flow, and verification. We are not replacing the reasoning loop — we are demoting it from the whole system to one component inside a scaffold that we engineer. The harness is what you build, specific to your task. A framework — LangChain, LangGraph, CrewAI — is tooling that helps you build a harness, or provides one off the shelf. The two are often conflated, and the difference matters more than it might seem at first. Anthropic's Claude Agent SDK frames the canonical harness loop as gather context → take action → verify work → repeat. The verify stage is a meaningful addition that classic ReAct does not have — it externalises judgment rather than asking the model to self-critique.
